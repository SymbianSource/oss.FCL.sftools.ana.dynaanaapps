Dynamic Binary Support Trace and Analysis

Dynamic binary support trace analysis is dedicated to performing function level performance analysis on binaries that have been uploaded to the device at run-time, at least in the operating system’s point of view. In those cases, the address of the binary is not known at compile-time. If sampled with any of the previously mentioned methods, it is impossible to tell the binary from an address that is not found from the symbol file, which is produced at run-time. One solution to resolve the problem is to store small pieces of the executed binary code at run-time and store them along with the trace taken.

Performance Investigator suppports EKA2 dynamic support trace and analysis. Dynamic binary support trace in EKA2 works on a different principle than in EKA1. This is facilitated by certain changes in the kernel. In the EKA2 version, the collected information provides a semantically similar result as the EKA1 version, thus information about dynamically loaded binaries and their locations in memory is provided as the result of the analysis. However, the information collected from the system is much more straightforward in the EKA2 version.

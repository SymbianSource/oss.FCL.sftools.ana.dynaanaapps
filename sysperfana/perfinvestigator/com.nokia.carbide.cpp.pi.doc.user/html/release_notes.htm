Release Notes

Performance Investigator Analyser Carbide.c++ Extension – Version 2.4.0

Released June 20th, 2010

  • What's new
  • Target Performance Investigator Agent
  • Fixed bugs and workarounds
  • Technical support

What's New

  • Graphics Memory Usage data support— Support for viewing graphics memory usage. Both private and shared graphics memory usage can be displayed in the graph.
  • Performance Investigator Wizard — Support for tracing Profiler data remotely from the device.
  • Performance Investigator View — Support for handling imported Profiler data files on the view.
  • Performance Counters Graph and Table — Support for viewing and analyzing the CPU clock rate.
  • CPU Thread Load Graph and Table — Support for scaling the thread load to the CPU Clock Speed.

Target Performance Investigator Profiler

  • To produce a Profiler data file, you need to install one of these profiler agents:
    • Carbide Profiler 2.0 Installer - for S60 3.0 (3rd Edition, initial release) devices
    • Carbide Profiler 2.0 (Dual CPU) Installer - for S60 3.1 (3rd Edition, Feature Pack 1) devices with dual CPUs, such as the N95 and N96
    • Carbide Profiler 2.0 (Single CPU)Installer - for S60 3.1 (3rd Edition, Feature Pack 1) devices with a single CPU, such as the E71, N76, 6290, N78 and 5320

    Those files and others can be can be downloaded and installed using the On-Device Connections > New Connection Wizard >Install remote agents pane.

Fixed bugs and workarounds

  • Complete list of Carbide.c++ bugs fixed
  • Workarounds for known issues:
Item Explanation
The Investigator menu item does not appear on the menu toolbar after an invalid format error occurs while opening an NPI file.

Workaround: To show the Investigator menu item, give focus to any non-NPI file or to any view other than the editor view (e.g., to the Symbian Project Navigator view).

[3005] Profiling an application with all traces enabled and using the maximum sample rates (50ms and 250ms) may cause Profiler to exhibit strange behaviors

Symptoms:

  • S60 - You may get unusual errors such loss of icons in lists.
  • UIQ 3.0 - The generated .dat file is corrupted and when imported into the Analyzer can cause a crash.

Workaround: Do not use maximum sampling rates, if all traces must be enabled, to avoid a high CPU overload condition. You can do this by either upping the time interval to 100ms/500ms or by disabling some traces.
[3049] Profiler sampling application closes when power usage sampling is requested on a device without power management

On devices, such the N93, that do not have power management hardware, attempting to sample power usage will fail, and the application that samples, CProfApp, will close.

There is no workaround.
Carbide.c++ may run out of memory If you open or import several large profiler data files, the Java Virtual Machine may run out of memory.

The workaround is to allocate more memory in the Carbide.c++.ini file (typically in the Carbide.c++ folder that contains Carbide.c++.exe). For example, to allocate 1GB, edit the Carbide.c++.ini to contain “-Xmx1024m” and then restart Carbide.c++.

NOTE The eclipse.ini file was renamed Carbide.c++.ini for purposes of this product.
Host cannot import debug output from the target device You can send sampling data to the debug port of your device. However, this release cannot import the .base64 file that contains this data. No workaround.

Version history

2.3.0

  • Performance improvements — Improved overall memory consumption.
  • Version numbers harmonized — Version numbers of Performance Investigator plug-ins and feature increased to 2.3.0.
  • Instruction trace (ITT) support — Support for the RVCT 4.0 map file format.
  • Library event support — Memory plug-in supports investigating memory usage by a library from the application developer's point of view.
  • Performance Investigator Import Wizard — Support for importing multiple profiler data files with the same configuration from a file system, and a new possibility to select the desired views from the available trace types for each imported file.

2.2.0

  • Interrupt support — Support for viewing software and hardware interrupts. Interrupts view can be used for tracking down low level performance issues.
  • Performance counter support — Support for viewing performance counter traces. This feature has been ported from Bappea to Performance Investigator. If the Instructions executed counter (performance counter number 7) has been activated during the profiling, the MIPS graph can be included in the Performance counter view.
  • Interconnect performance counter support — Support for viewing interconnect performance counter traces. This feature has been ported from Bappea to Performance Investigator.
  • SMP support for CPU Profiler — The Threads view can now be broken down into separate graphs, one per each available CPU on SMP systems.
  • Usability enhancements for graphs — Title bar added for each graph for manipulating the location and size of a single graph.
  • Version numbers harmonized — Version numbers of Performance Investigator plug-ins and feature increased to 2.2.0.
  • Instruction trace (ITT) support — Support for event based ITT traces (ITT_V2.00).

1.5.0

  • PI Profiler 2.02 support added. — Support for more detailed event based memory model added.
  • Touch Event support — Button plug-in is now able to recognize touch events.
  • Some minor usability improvements and bug fixes

1.3.1

  • Copy tables and paste into spreadsheets — copy displayed information in Threads, Binaries, Functions, and Function Calls tables to the clipboard and paste into a spreadsheet application, such as Microsoft Excel
  • Save tables — save information displayed in tables into a comma delimited .csv file format that can be opened in a spreadsheet application
  • Save more precise data in tables — save specific information about selected data within a table. For example, save samples for Threads that are checked in the Thread table or save the priority setting for Threads that are checked. You can also save data for one or more selected binaries, checked functions in Functions table, a single selected function in Function Calls table, checked memory samples, all button events for specified time interval, and all Power samples for time interval.

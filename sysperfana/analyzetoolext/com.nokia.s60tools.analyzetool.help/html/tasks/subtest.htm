Running subtests

You can run subtests which are tests inside a test run. While the test run life cycle is the same as for the tested process, the subtest is started and stopped inside the time frame of the test run. The number of test runs is not limited and they can overlap each other.

To run subtests:

  1. When the data capture is running and you have started the test run, you can start a new subtest by selecting Start subtest from the AnalyzeTool view. The subtest name dialog is now displayed, and starting from this point AnalyzeTool collects separate memory leak data for the subtest timeframe.

    When the subtest is started, the information is displayed in the message area. The information contains the target name and the subtest name.

  2. When the desired subtests in the target device are run, the next step is to end the active subtest by selecting Stop subtest from the AnalyzeTool view.

    If there are multiple subtests running, the Select subtest to stop dialog is displayed.

    Information of ended subtest is displayed in the message area.

When the test run is finished, AnalyzeTool shows the data about subtests run under the test run.

Viewing memory statistics

Memory usage statistics are generated automatically after the data capturing is stopped. Alternative to the data capture user can open the existing trace file to see memory statistics. For more information, see AnalyzeTool files.

There are two options for opening a memory analysis data file:

  • From Project pop-up menu, select AnalyzeTool > Open and analyze data file
  • In the AnalyzeTool view, select Open and analyze data file

Note! The memory statistics generation must be activated from the AnalyzeTool preferences page, see AnalyzeTool preferences

The memory statistics are displayed in the AnalyzeTool window.


Description of statistics tab
  • File column: Source code file name.
  • Function column: Function name.
  • Line column: Source code line number.
  • Allocations column: How many allocations match to the source code location.
  • Time column: The last allocation time.
  • Size column: Total size of allocations.

User can select/change active run by using drop down list. By default the statistics information is ordered by “Allocations” column. The code line matching most allocations is topmost of the list. User can reorganize the list by clicking “Allocations”, “Time” or “Size” columns.

Double-clicking the statistic item opens the appropriate source code file to that code line.

TraceViewer basic steps

Open the TraceViewer tool in either of the following ways:

  • From the Carbide > TraceViewer menu.
  • From the Window > Show view > Other > Carbide extensions menu.

Tracing a target device involves the following stages, some of which are optional.

  1. Set up tracing for your development goals.
    • Setting TraceViewer preferences (Connection settings are mandatory to define)
    • Configuring triggers (optional)
    • Configuring logging for traces (optional)
    • Setting up rules and filters (optional)
    • Decoding binary traces with a Dictionary file(mandatory for OST traces)
    • Activating traces(mandatory for OST traces)
  2. Connect to the device to start receiving traces.
    • Connecting to the trace source.
  3. Start the device, the application or the test case you are working on.

    The received traces should be visible in the TraceViewer main view, as shown below in an example.

  4. Analyse the data to get more information of the trace itself or open the trace location in a text editor (only for OST traces).
  5. (Optional) You can also copy the trace information to the clipboard or search for traces.

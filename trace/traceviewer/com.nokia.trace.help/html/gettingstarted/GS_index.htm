Getting started

The topics in this section help you understand the basic functionality and workflow in using the Tracing tools.

For a basic walk-through of OST tracing, see Basic walk-through of tracing.

Click the topics available on the left-hand navigation pane to find tool-specific introductions.

   TraceBuilder

A tool for adding Open System Traces (OST) to the source code. TraceBuilder consists of two components:

  • TraceCompiler that parses OST Application Programming Interface (API) format from the source files and generates the corresponding decode files.
  • TraceBuilder UI that enables you to add and maintain OST traces.
   TraceViewer

A tool for activating, viewing and manipulating traces received from a target device.

   TraceAnalyser

TraceAnalyser is a tool that enables you to measure the time consumption of an application and detect timing violations. It also enables you to supervise parameters in Open System Trace (OST) traces.

Tracing requires also the following non-Carbide tools:

  • TraceCompiler – build time tool used to generate the needed headers and decoding files for traces.
  • TraceSwitch – Symbian application used to route traces either to USB or to a file.

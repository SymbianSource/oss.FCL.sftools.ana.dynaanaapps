Setting project include paths and compiling

To use Open System Trace (OST) traces, you must include the OpenSystemTrace.h header file from the environment. Currently TraceBuilder automatically adds the necessary system include macro to the MMP file if it does not exist.

If you get an error about a missing header when compiling, set the project's include paths:

  1. Search the /epoc32/include/ folder for the OpenSystemTrace.h file.
  2. Then add a following line to your component's MMP file:
    SYSTEMINCLUDE <the path where OpenSystemTrace.h was found.

    Note: If OpenSystemTrace.h was not found from your development environment, you cannot use OST traces.

  3. Compile your component using OST traces.

    The compilation is handled by Trace Compiler.

Note that you might also get errors such as Userinclude path "/traces/" not found or Can't find following headers in User or System Include Paths "OstTraceDefinitions.h" when you are compiling for the first time. This is because those paths and files are generated by TraceCompiler, and in the first time they do not exist when the Symbian Build System generates makefile.

When TraceCompiler is run to a component containing OST traces, you should see this kind of message:

******************* TRACECOMPILER STARTS *******************
Building traces for component Hello3. Component UID: 0xE1B04078.
******************* TRACECOMPILER ENDS *********************

After compilation, you should find a generated dictionary file for your component in the /epoc32/ost_dictionaries/ folder.

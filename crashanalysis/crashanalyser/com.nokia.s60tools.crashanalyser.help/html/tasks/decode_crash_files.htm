Decoding crash files

You can both decode undecoded files and redecode already decoded files with the Decode files functionality. Note! You cannot decode emulator panic files, and also some decoded files cannot be redecoded because their original binary file is not necessarily available. If you try to decode files that are not decodable, an error message will be displayed.

To decode crash files:

  1. Select decodable files from the Crash Analyser main view.
  2. Click the Decode files button.
  3. Define the location of symbol, map, and image files. You have to define either the symbol files or map files or both of them. Defining image files is optional. For more information, see Symbol files, Map files, and Image files.
  4. Begin decoding by selecting Finish.
  5. Decoded files are displayed in the Crash Analyser main view. You can start analysing the files, see How to analyse a crash.

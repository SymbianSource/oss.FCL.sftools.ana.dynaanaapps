Importing crash files

  1. Activate the Crash Analyser Import Wizard in the Crash Analyser view by clicking the Import files button.
  2. Select a crash file or the folder where you have crash files or press Read files from device button to fetch files directly from a device which is connected via PC Suite connection.

    To select the files to be analysed, you must first input the folder where the crash files are found. The Crash Analyser Import Wizard remembers all previous input folders and files, suggesting the most recently used as the default. To specify the directory or file, either select a previously used directory from the drop-down list, or type in a new directory path.

    Note! It takes time to decode files, so do not select too many files to be decoded at a time.

  3. From the file table, select files to be analysed.

    When you click Next, the crash files found in the given directory are displayed in the file table. The files are pre-parsed for their image information, which can be viewed in the File info text field by highlighting the desired file.

    Note! You cannot select two files that are not from the same image.

    Note! If you are importing decoded files (.crashxml files), this is the last step of the Import Wizard.

  4. Define the location of symbol, map, and image files. You have to define either the symbol files or map files or both of them. Defining image files is optional. For more information, see Symbol files, Map files, and Image files

    Begin importing by selecting Finish.

  5. Imported files are displayed in the Crash Analyser main view. You can start analysing the files, see How to analyse a crash.
